**親 SEQA**: SEQA-<AREA>-NNN

> **記述規律**: RBA のドメイン主語をクラスに mapping し、クラス境界・属性・操作を識別する。**操作名は人間の言語**。関数名(言語命名規則)・引数型・戻り型・修飾子・言語固有メカニズムは書かない。詳細は `04-iconix-layer.md` §5。

## クラス一覧

### Boundary クラス

```
注文受付窓口
─────────
- 接続コンテキスト : 接続情報
─────────
+ 注文を投稿する(顧客, 注文項目のコレクション) : 結果
+ 確定通知を返す(注文識別子) : 結果
─────────
<<boundary>>
```

### Control クラス

```
注文確定処理
─────────
(属性なし、ロジック保持)
─────────
+ 注文を確定する(顧客, 注文項目のコレクション) : 結果
+ 注文をキャンセルする(注文識別子, 理由) : 結果
─────────
<<control>>
```

```
在庫引当処理
─────────
(属性なし、ロジック保持)
─────────
+ 在庫を引当てる(注文項目のコレクション) : 引当結果
+ 引当を解除する(引当識別子) : 結果
─────────
<<control>>
```

### Entity クラス

```
注文
─────────
- 識別子 : 識別子
- 状態 : 注文状態
- 顧客 : 顧客への参照
- 項目 : OrderItem のコレクション
- 確定日時 : 日時
─────────
+ 注文を保存する() : 結果
+ 注文を取り出す(識別子) : 注文
─────────
<<persistent>>
```

```
在庫
─────────
- 商品識別子 : 識別子
- 在庫数 : 数値
─────────
+ 在庫を確認する(商品識別子) : 在庫数
+ 在庫を減らす(商品識別子, 数量) : 結果
─────────
<<persistent>>
```

> **記述例の注意**: 上記の `識別子`, `数値`, `日時`, `コレクション`, `〜への参照` は **概念型**(言語非依存)。Rust の `i64`, `String`, `Vec<T>`, `Arc<T>` などは書かない。これらは DD で確定する。

## クラス間の関係

| 関係元 | 関係先 | 種類 | カーディナリティ | 説明 |
|---|---|---|---|---|
| 注文 | 顧客 | aggregation | 多対 1 | 注文は顧客に紐付く(顧客は注文の所有者ではない) |
| 注文 | OrderItem | composition | 1 対多 | 注文は OrderItem を所有する |
| 注文確定処理 | 在庫引当処理 | 依存 | — | 注文確定処理は在庫引当処理を利用する |
| 在庫引当処理 | 在庫 | 依存 | — | 在庫引当処理は在庫を参照・更新する |

## ステレオタイプ凡例

- `<<boundary>>`: アクター・外部システムとの境界
- `<<control>>`: 振る舞いの制御ロジック
- `<<persistent>>`: 永続化対象
- `<<external>>`: 外部依存(別システム)
- `<<concurrent>>`: 並行に動作する主体(概念レベル、具体機構は DD)

## 通信制約遵守チェック(具体側)

- [ ] Boundary 同士の直接通信なし
- [ ] Entity 同士の直接通信なし
- [ ] Boundary → Entity 直結なし
- [ ] Actor → Control / Entity 直結なし

## レイヤ汚染チェック

- [ ] 関数名(snake_case/camelCase の関数呼び出し)が含まれていない
- [ ] 引数型・戻り型(`Result<T,E>`, `Vec<T>`, `&str` 等)が含まれていない
- [ ] 修飾子(`pub`, `async`, `static`)が含まれていない
- [ ] crate/package 識別子(`tokio::`, `std::`)が含まれていない
- [ ] 操作名は人間の言語(自然言語、または PascalCase/概念名)で書かれている

違反が見つかったら、該当箇所を DD に移動する。
